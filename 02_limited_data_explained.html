The 32-Label Challenge

How far can a model get when it is only allowed to learn from 32 labeled examples? 2a & 2b are run and scored; 2c is partly run; 2d & 2e are designed.

02GROUND RULES

The rules of the game

Fixed for all of Part 2
  • 32 labels only. Just the 32 sampled examples count as labeled training data for the BERT arms (2a, 2b, 2d). The ~64k other train rows are "unlabeled" — in-bounds only for the planned 2e self-training idea.
  • Tune on validation, touch test once. Every choice is made on validation; a finished model is frozen, then run on test exactly once. No studying for the exam.
  • One fixed recipe, no hyperparameter search for 2a — the few-sample stability recipe is adopted as-is; validation is used only for two sanity checks, and that single use is disclosed.
  • SEED = 42 everywhere. The 32-example draw, the splits, and the bootstrap CIs are all seeded; the 32 are drawn once and frozen.
  • Suicidal recall reported every time, even when it craters.
  • Heavy training on Colab, scoring at home, so every result file is stamped with the correct provenance.

04HOW IT WORKS

How the notebook is built

Colab trains, your laptop scores
  • Training cells run on Colab (free T4 GPU). They fine-tune BERT and write raw predictions only — a CSV of test_id → predicted label — into staging (results/colab_runs/). Colab never sees test labels and never computes a score.
  • Scoring cells run locally. They read those CSVs and run the shared harness, which stamps the local split fingerprint and writes the official files under results/metrics/, predictions/, and figures/.
  • One reusable trainer. A single train_and_predict() does every arm; only train_df changes between experiments, so the recipe is byte-identical across arms — that is what makes the comparisons fair.
The 2a recipe, and why these knobs

bert-base-uncased · lr 2e-5 · batch 8 · 40 epochs (~160 steps) · 16 warmup steps · weight decay 0.01 · max length 256 · mixed precision.

  • Why 40 epochs, not 3? 32 examples at batch 8 is only 4 optimizer steps per epoch; 3 epochs = 12 updates, too few — the model collapses to one or two classes. The tiny-data fix is a small learning rate plus ~10× longer training to near-zero loss. A tripwire warns if final training loss isn't below 0.1.
  • Why 256 tokens? Covers ~91% of posts; 512 would double test-inference cost for ~5 pp more coverage.

Dataset augmentation (1.0 pt)

What we did
  • Arm B — back-translation (primary): translate each example to another language and back (de, fr, es, ru) to get paraphrases that keep meaning but change wording.
  • Arm C — EDA (size-matched comparison): simple word-level edits, with crisis words (die, kill, suicide) and negators (not, never) protected so the label can't flip.
  • Both grow the 32 to ~150 examples and retrain with the identical 2a recipe. Same size → the B-vs-C comparison isolates augmentation quality, not quantity.

Zero-shot LLM (0.5 pt)

The idea
  • Don't train anything — just ask an LLM to classify each post with no examples. Model: Gemini 2.5 Flash (free tier, $0), temperature 0, forced to answer with one of the 7 exact label strings.
  • Why a subsample: running on all 20k rows is rate-limited and slow. A pre-registered proportional subsample of 2,100 test rows keeps the test imbalance and is declared the final, only 2c number in advance — no "run more if it looks good."
Done so far (steps 1–2)
  • Subsample frozen — 2,100 IDs, fingerprint efa3a4ff…. Per-class: anxiety 350, normal 335, depression 315, stress 312, personality disorder 284, bipolar 282, suicidal 222.
  • Prompt frozen — system prompt with a one-line definition per label and an explicit active ideation vs. low mood boundary; hash 165799bb…, google-genai 1.68.0.
Pending (steps 3–5)
  • Develop the prompt on ≤200 validation rows, then freeze (started, not finished).
  • Run the 2,100 rows once as a resumable, rate-limited loop (~3.5 h at 10 req/min).
  • Score on the manual subset path + McNemar vs the mined lexicon on the same 2,100 IDs.
Two honesty rules: refusals are retried once then scored incorrect (never dropped — dropping inflates suicidal recall); and tuning the prompt on validation is a disclosed deviation, so the 2c number is framed as an optimistic bound.

11GUARDRAILS

Reproducibility & safety

How a result can't quietly drift
  • Three fingerprint guards — the split, the 32-example sample, and the 2c subsample each recompute a SHA-256 and refuse to run (or overwrite) if it changes.
  • Pinned everything — dataset revision, library versions (recorded in each JSON), SEED = 42 throughout.
  • Frozen-once generation — augmented (and later synthetic) data is generated once, then guarded against regeneration.
  • Silent-failure tripwire — a run whose training loss didn't drop near zero is flagged before its numbers are trusted, so a failed run can't masquerade as "BERT can't learn from 32."
  • Idempotent staging — a seed/arm whose prediction CSV already exists is never silently retrained.
12Q&A BANK

Questions and answers

Why 40 epochs instead of the usual 2–3?
With 32 examples at batch 8, one epoch is only 4 optimizer steps; 3 epochs = 12 updates, too few, so the model collapses to one or two classes. The tiny-data stability fix is a small learning rate plus ~10× longer training to near-zero loss.
Isn't training 3 seeds and reporting only seed 42 cherry-picking?
No — seed 42 was declared canonical before any results existed. All 3 seeds are reported (mean ± std); only the pre-declared one enters the official ladder. The huge suicidal-recall spread (0.109 → 0.405) is exactly why we report variance instead of one lucky number.
Augmentation raised accuracy — why call back-translation a problem?
Because it lowered suicidal recall (0.129) below the 32-only baseline (0.163). It improves the easy majority classes while losing the hardest, most ethically important one. EDA, with lower accuracy, did the opposite (0.272). The size-matched B-vs-C comparison isolates this quality difference.
Why only 2,100 rows for the LLM, and why fix it in advance?
The free-tier rate cap makes 20k rows impractical (~3.5 h even for 2,100). A proportional 2,100 keeps the test imbalance and gives a ±1.9 pp accuracy CI. Pre-registering it as the only 2c number prevents an "if it looks good, run more" bias.
Why are LLM refusals scored as wrong instead of dropped?
Refusals cluster on suicidal content. Dropping them would remove the hardest suicidal cases and artificially inflate suicidal recall — the opposite of honest.
How is this comparable to Part 1 at all?
Identical frozen test set, verified by fingerprint before every run; same metrics, same harness. The only thing that changed is how much labeled data the model was allowed to learn from.
13GLOSSARY

Terms used above, in one breath each

AccuracyShare of test posts whose predicted class is correct. Easy to inflate on imbalanced data, so never read alone.
PrecisionOf the texts the model called class X, how many really were X?
RecallOf the texts that really are class X, how many did the model catch?
F1 / Macro-F1F1 = harmonic mean of precision and recall. Macro-F1 averages it per class with equal weight, so big classes can't mask failure on small ones.
Suicidal recallOf all genuinely suicidal posts, the share the model flags. The ethics-critical metric; missing these is the costliest error.
Cohen's kappa (κ)Agreement with the truth after subtracting chance agreement. 0 = guessing, 1 = perfect; always-majority scores 0.
Bootstrap 95% CIResample the predictions many times to estimate the range the true score likely sits in. Non-overlapping CIs mean a difference is real, not noise.
McNemar testA paired significance test on two models' right/wrong patterns over the same examples — "is A really better than B?"
Zero-shotAsking a model to classify with no training examples, purely from instructions.
Back-translationParaphrasing by translating a sentence to another language and back.
EDAEasy Data Augmentation — simple word-level edits (synonym swap, random insertion/deletion/swap).
Prompt / system promptThe instructions an LLM is given at inference time. The system prompt sets the standing rules (here: label definitions + output format); it teaches nothing, it points at what the model already knows.
TemperatureHow much randomness an LLM uses when picking the next word. 0 = always the most likely choice — what you want for grading, not for creative writing.
Pseudo-labelingLet a model label the ~64k unlabeled rows itself, then train on its own confident guesses. Cheap labels, but errors can snowball — a 2e candidate.
SetFitA few-shot training recipe built for tiny label budgets: first teach a sentence-encoder to pull same-class texts together (contrastive pairs squeeze more signal from 32 examples), then fit a small classifier on top — a 2e candidate.
SeedThe number that fixes all randomness so a run is reproducible. SEED = 42 here.
Fingerprint (SHA-256)A hash of a dataset; if one row changes, the hash changes, so a guard can detect drift.
14CONCEPT

Understanding BERT — plain English and technical

The one-sentence version

BERT is a neural network that already learned how English works by reading an enormous amount of text — and in Part 2 we nudge that knowledge toward our 7-class task. We don't teach it English from scratch (we couldn't, with 32 examples); we take a model that already understands language and fine-tune it.

Definition. BERT = Bidirectional Encoder Representations from Transformers. Bidirectional — it reads the whole sentence at once, using both left and right context, not word-by-word in one direction. Encoder — it turns text into rich numeric meaning-vectors. Transformers — the architecture that makes this work, built on self-attention (below).
Why it beats keywords: context

Plain English. A word like "die" means different things in different sentences. The older approach (Word2Vec / GloVe) gives every word one fixed vector, so "die" always looks the same. BERT gives each occurrence its own vector, shaped by the words around it.

Technical. These are contextual embeddings: the same token gets a different representation depending on its neighbours. That is the entire reason we moved from Part 1's lexicon to BERT — "I want to die" and "I want to die laughing" share three words and mean opposite things; a keyword/lexicon model can't separate them, BERT can.

How a post flows through BERT
  1. Tokenize (WordPiece). Split the text into subword tokens and add a special [CLS] at the front and [SEP] at the end. Rare words split into pieces (unhappiness → un ##happi ##ness). Hard cap 512 tokens — this notebook uses 256.
  2. Embed. Each token becomes a starting vector that encodes both which word it is and where it sits in the sentence.
  3. 12 transformer layers. Each layer runs self-attention (next block) + a small feed-forward step, so meaning compounds as it passes upward.
  4. The [CLS] vector. After the layers, the [CLS] token's final vector (768 numbers) is treated as a summary of the whole post.
  5. Classification head. One small linear layer turns those 768 numbers into 7 scores (logits); softmax turns scores into probabilities; the largest wins.
Self-attention, the core idea

Plain English. Each word gets to "look at" every other word and decide which ones matter for its own meaning. "die" looks at "laughing" and shifts toward the idiom instead of the literal sense.

Technical. For every token, attention computes a weighted average of all the tokens' vectors. The weights come from comparing a token's query with every token's key (a dot product, then softmax), so "how much should I listen to you?" is learned. "Multi-head" means it does this several times in parallel to capture different kinds of relationships (syntax, topic, negation), and stacking 12 layers lets the meaning build up.

Two phases: pre-training vs fine-tuning
Pre-training
Done once by Google on Wikipedia + books. Masked language modelling: hide ~15% of words and predict them. This forces a deep grasp of grammar and meaning. We just download the result.
Fine-tuning
What we do in 2a. Keep those learned weights and keep training on our 32 labeled examples, so general English understanding gets specialised to "which of 7 conditions."
One way to picture it: pre-training is years of reading; fine-tuning is a short course on our specific labelling job.
Why 32 examples make fine-tuning unstable. bert-base-uncased has ~110 million parameters. Adjusting that many weights from only 32 examples is hugely under-determined — small changes in random seed or number of steps swing the outcome, and the model can collapse to predicting one class. That is exactly why 2a uses a small learning rate, ~40 epochs to near-zero loss, and reports 3-seed variance instead of a single number (recall the suicidal-recall spread 0.109 → 0.405).

15PRESENT IT

The methods, conceptually — the ideas behind 2a–2e

IN ONE LINE Every Part 2 experiment is the same question asked a different way: when you only have 32 labels, where can the rest of the knowledge come from?
One map of all five methods
2a — knowledge from pre-training.
BERT arrives having already "read" Wikipedia + books; the 32 labels only steer that knowledge toward our 7 classes. We're not teaching English, we're giving a short course.
2b — knowledge from invariance.
Stretch 32 examples by teaching what doesn't matter: "this rephrasing keeps the same label." Each augmented copy is a lesson in ignoring surface variation.
2c — knowledge rented.
Train nothing. Borrow a giant model (Gemini) that already studied this kind of language, and just ask. Zero of our labels are used for learning.
2d — knowledge distilled.
Make the rented knowledge permanent: have the giant model write practice examples, then train our small BERT on them. The teacher's knowledge gets compressed into a model we own.

2e then combines whatever worked — chosen on validation evidence only. The arbiter never changes: the same frozen test set, with suicidal recall deciding ties.

Why zero-shot works at all (2c)
  • The prompt doesn't add any knowledge — it points at knowledge the model already has. During pre-training the LLM read millions of posts about anxiety, depression, and crisis; our system prompt just tells it which of its existing concepts to retrieve and what output format to use.
  • Analogy: a med student who has read every textbook but never worked a shift at your clinic. They can still triage — not because you trained them, but because the patterns transfer.
  • That's also why the knobs are set to "exam mode": temperature 0 (always pick the most likely answer, no creativity) and a forced choice over the 7 exact labels (no essays, no hedging, no 8th class).
  • And it's why 2c can plausibly beat models we did train: Gemini's pre-training dwarfs anything 32 labels can teach BERT. The interesting question isn't "is the giant model good?" — it's "by how much, and at what cost?" (next block).
The shot spectrum — and a naming trap
Naming trap — "one-shot" means two unrelated things in this project. In LLM vocabulary, one-shot = one example included in the prompt. In our evaluation protocol, a "one-shot test run" = the frozen test set is touched once. 2c is zero-shot prompting evaluated in a one-shot run. If an examiner says "one-shot", ask which one they mean.
LLM-generated data is distillation (2d)
  • The core idea: the teacher writes practice problems; the student trains on them. The giant model's knowledge gets compressed into data, and the data trains a model we own.
  • Why bother, if 2c can just ask the teacher directly? Cost, speed, privacy, control: classifying every future post via a rate-limited external API (~3.5 h for 2,100 rows) vs a 110M-parameter model that runs locally, instantly, on sensitive text that never leaves the machine. 2d tries to keep 2c's knowledge without 2c's dependency.
  • The risk — a photocopy of a photocopy: synthetic text tends to be too clean, too repetitive, and subtly off-distribution from real posts. Train on it naively and BERT learns the teacher's writing style, not the disorder. That is exactly what the 3-stage quality gate is for: dedup kills repetition, independent relabeling kills label noise, and the length/vocabulary check kills style shortcuts.
  • The built-in finding: arm D (synthetic-only) vs arm C (32 real + synthetic) measures what the 32 real anchors are worth — and the gap between arm C and real-data training measures what synthetic data can't replace.
Augmentation is teaching invariance (2b)
  • Every augmented copy is a sentence in disguise: "this variation does not change the label." Synonym swaps, paraphrases, translation round-trips — all ways of saying "ignore the surface, keep the meaning."
  • The trap is teaching a false invariance. Back-translation smooths wording — but for suicidal vs depression, the wording is the signal (severity, negation, active vs passive ideation). Round-tripping "I can't go on" into softer phrasing tells BERT those cues don't matter. They do: suicidal recall fell to 0.129, below even the 32-only run (0.163).
  • EDA's clumsy word-level edits preserve crisis words by design — a truer invariance for this task — and suicidal recall nearly doubled to 0.272 despite lower overall accuracy.
  • One sentence to carry into the defense: an augmenter is only as safe as the invariance it teaches. That's why 2b's "winner" depends on the objective.
Say this in the defense: "Part 2's methods are five answers to one question: where does knowledge come from when labels are scarce? From pre-training, steered by 32 labels (2a); from invariances that stretch those labels (2b); rented from a giant model at inference time (2c); or distilled from that model into one we own (2d). The same frozen test set arbitrates all of them, and suicidal recall — the cost of a miss — decides what 'better' means."
